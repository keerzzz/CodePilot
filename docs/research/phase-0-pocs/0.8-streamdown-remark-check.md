# 0.8 Streamdown `plugins` / `remarkPlugins` 兼容性核查

> 目标：判断 Phase 5.2「按 heading 折叠」是否可以用一个标准 unified/remark 插件（`remarkCollapsibleSections`）直接塞进 Streamdown。

## [外部事实] Streamdown 组件签名

`node_modules/streamdown/dist/index.d.ts:18-24` 中 `Options`（被 `StreamdownProps` extend）定义：

```ts
interface Options {
  children?: string;
  components?: Components;
  rehypePlugins?: PluggableList;   // ← 标准 unified 插件列表
  remarkPlugins?: PluggableList;   // ← 标准 unified 插件列表
  remarkRehypeOptions?: Readonly<Options$1>;
}
```

`index.d.ts:146-151, 196` 里另有 `plugins?: PluginConfig`：

```ts
interface PluginConfig {
  code?: CodeHighlighterPlugin;   // { name:"shiki", type:"code-highlighter", highlight, ... }
  mermaid?: DiagramPlugin;        // { name:"mermaid", type:"diagram", language, getMermaid }
  math?: MathPlugin;              // { remarkPlugin: Pluggable, rehypePlugin: Pluggable }
  cjk?: CjkPlugin;                // { remarkPluginsBefore, remarkPluginsAfter, ... }
}
```

结论：Streamdown **同时存在两条插件通道**：
- `remarkPlugins` / `rehypePlugins`：承接原生 unified `PluggableList`
- `plugins`：承接 Streamdown 自定义对象链（code/mermaid/math/cjk 四个专用槽），**不是** 通用 remark 挂载点

## [外部事实] Streamdown 内部消费逻辑

`node_modules/streamdown/dist/chunk-5FQGJX7Z.js`（压缩产物）中组装 remark 链的片段：

```js
R=useMemo(()=>{
  let S=[];
  if(a?.cjk)  S=[...S,...a.cjk.remarkPluginsBefore];
  S=[...S,...s];                    // s = props.remarkPlugins
  if(a?.cjk)  S=[...S,...a.cjk.remarkPluginsAfter];
  if(a?.math) S=[...S,a.math.remarkPlugin];
  return S;
},[s,a?.math,a?.cjk]);
// 随后 <Ie remarkPlugins={R} rehypePlugins={Y} .../>  （Ie 为 react-markdown）
```

即 Streamdown 把 `props.remarkPlugins`（`s`）与 `plugins.cjk/math` 中抽出的 `Pluggable` 拼成最终列表，整体喂给内部 `react-markdown`，由后者调 `unified().use(...)`。`plugins.code` / `plugins.mermaid` 走的是 components 渲染通路（`StreamdownContext` 提供给代码块/图表组件），不进 remark AST。

子插件形状验证 `node_modules/@streamdown/math/dist/index.js:1`：

```js
function g(e={}){
  let a=[o,{singleDollarTextMath:...}];   // o = remark-math
  let l=[n,{errorColor:...}];             // n = rehype-katex
  return {name:"katex",type:"math",remarkPlugin:a,rehypePlugin:l,getStyles(){...}};
}
```

`remarkPlugin` / `rehypePlugin` 字段就是标准 `Pluggable`（`[plugin, options]` 元组），子插件只是把标准 remark 插件"打包"到对象里供 Streamdown 展开。

## [结论] 能接 —— 但走的是 `remarkPlugins` 通道，不是 `plugins`

- `remarkCollapsibleSections` 作为标准 `function (): Transformer { return (tree)=>... }` 可以直接放进 `<Streamdown remarkPlugins={[remarkCollapsibleSections]} plugins={_streamdownPlugins} ... />`
- 不要尝试塞进 `plugins` 对象 —— 那里只认 `code/mermaid/math/cjk` 四个具名槽，自定义 key 会被忽略
- 现有 `src/components/layout/panels/PreviewPanel.tsx:30-35, :397` 已通过 `plugins={_streamdownPlugins}` 注入四件套，新增 `remarkPlugins` prop 不冲突，两条通道并行

实作注意：
1. `plugins.cjk.remarkPluginsBefore` 会排在 `props.remarkPlugins` **之前**，`...After` 和 `math.remarkPlugin` 排在之后。若折叠插件需要在 heading 结构稳定后再运行，放 `remarkPlugins` 末尾即可（math 在它后面但只处理公式节点，无冲突）
2. `mode="streaming"` 下 Streamdown 按 block 切片渲染（`parseMarkdownIntoBlocks`），跨 block 的 heading→content 关系在流式阶段可能被切断 —— 折叠插件需对单 block 也要鲁棒（空 section / heading-only block 不报错）
3. `rehypePlugins` 同理可用，若选 AST 层包 `<details>`，可直接在 remark 层把 heading + 后续 sibling 包成 MDAST 自定义节点，或在 rehype 层包 HAST

## [降级方案]

本案**无需降级**。原计划 5.2 直接以 `remarkPlugins` 落地即可，Phase 5.2 决策：采用路径 A（标准 remark 插件），不必启用 DOM post-processor / 切换 react-markdown / patch Streamdown。
