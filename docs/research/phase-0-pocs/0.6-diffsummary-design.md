# Phase 0.6 POC — DiffSummary 升级为 Artifact 入口

## [仓库事实]

- 内联组件定义：`src/components/chat/MessageItem.tsx:535-572`，函数签名 `function DiffSummary({ files }: { files: Array<{ path: string; name: string }> })`，非独立文件。
- 渲染调用点：`src/components/chat/MessageItem.tsx:712-728`，从 `pairedTools` 过滤 `WRITE_TOOLS = { write, edit, writefile, write_file, create_file, createfile, notebookedit, notebook_edit }`（`MessageItem.tsx:714`），抽 `file_path | path | filePath`（`:719`），去重后传入 `<DiffSummary files={unique} />`（`:727`）。
- 仅此一处使用（grep `DiffSummary` 只命中 L535 定义和 L727 调用）。
- DiffSummary 内部状态：单个 `const [open, setOpen] = useState(false)`（`MessageItem.tsx:536`），展开时 `motion.div` 列出每个 `f.name`（`:561-564`）。
- ToolActionsGroup 已独立于 `src/components/ai-elements/tool-actions-group.tsx:500-628`，write/edit 系工具通过 `TOOL_REGISTRY` match 规则（`tool-actions-group.tsx:126-133`）渲染为一行 `Edit <filename>`，**与 DiffSummary 呈现的是同一批文件**。
- DiffSummary 使用的图标/动画依赖（`CaretRight`, `NotePencil`, `motion`, `cn`）均已在 MessageItem 顶部导入（`MessageItem.tsx:5, 15`），抽出后需同步导入。
- 渲染位置紧挨 `<MessageContent>`（`:710` 收口），位于 footer（`:731`）之前；与 ToolActionsGroup（`:650-661`）在 MessageContent 内部，DiffSummary 在 MessageContent 外部兄弟节点。

## [分析]

### DiffSummary 耦合度（低）
- 仅依赖 `files` prop，**不访问** `message / sessionId / isAssistantProject / isUser / displayText` 等 closure 变量。
- 仅用本地 `useState(open)`，未读 MessageItem 的 `isExpanded / isOverflowing / contentRef / tokenUsage`。
- 不调用 MessageItem 内定义的 helper（`parseToolBlocks / pairTools / parseMessageFiles` 都在 import 之外）。
- 结论：**完全可抽成独立文件**，无破坏性风险。抽出后 `MessageItem.tsx:535-572` 删除，`:727` 改为 `import { DiffSummary }`。

### 与 ToolActionsGroup 职责边界（独立、互补）
- ToolActionsGroup 以**时序+状态**呈现所有工具动作（running / success / error，含 read/grep/bash），`tool-actions-group.tsx:455-482` 强调单行信息密度。
- DiffSummary 只聚焦**最终落盘的写入文件清单**（已过滤 `!t.isError`，`MessageItem.tsx:716），去重（`:726`），是一个"这轮消息产出了什么文件"的总览。
- 二者不冲突：同一次 edit 工具会在 ToolActionsGroup 出现一行"Edit xxx.md"，同时在 DiffSummary 下方作为成果摘要。预览/导出按钮逻辑上只属于"产出物"，放 ToolActionsGroup 行内会污染通用工具渲染链路（需改 `TOOL_REGISTRY` 契约引入 actions），放 DiffSummary 是单点最小侵入。
- 结论：**职责互不影响**，按钮入口应落在 DiffSummary，不应合并。

## [决策与改造接口]

### 决策
1. **抽独立组件** → 新文件 `src/components/chat/DiffSummary.tsx`（耦合度为零，且 Phase 后续需在外部复用/单测）。
2. **按扩展名分级能力**，组件内判定，不向上泄漏逻辑。
3. **与 ToolActionsGroup 保持独立**，不改其 registry。

### TS 接口契约

```ts
// src/components/chat/DiffSummary.tsx
export type DiffFile = { path: string; name: string };

export interface DiffSummaryProps {
  files: DiffFile[];
  /** 点击"预览"，宿主负责打开 artifact 面板 */
  onPreview?: (file: DiffFile) => void;
  /** 点击"导出长图"，宿主负责渲染 + 下载 */
  onExportLongShot?: (file: DiffFile) => void;
}
```

### 扩展名 → 按钮可见性规则

| 扩展名（小写，取自 `name` 末尾 `.xxx`） | 预览 | 导出长图 |
|---|---|---|
| `.md` / `.mdx`                         | 显示 | 不显示 |
| `.html` / `.htm`                       | 显示 | 显示 |
| `.jsx` / `.tsx`                        | 显示 | 显示 |
| 其他（`.ts/.js/.py/.json/...` 等）     | 不显示 | 不显示（保留原 diff 摘要） |

辅助函数（组件内私有）：
```ts
function getExt(name: string): string {
  const i = name.lastIndexOf('.');
  return i >= 0 ? name.slice(i).toLowerCase() : '';
}
const PREVIEWABLE = new Set(['.md', '.mdx', '.html', '.htm', '.jsx', '.tsx']);
const LONGSHOT    = new Set(['.html', '.htm', '.jsx', '.tsx']);
```

按钮渲染条件：
- `onPreview && PREVIEWABLE.has(ext)` → 显示"预览"。
- `onExportLongShot && LONGSHOT.has(ext)` → 显示"导出长图"。
- 回调未传时一律不渲染按钮（保证宿主未接线时零视觉变化，便于分阶段上线）。
